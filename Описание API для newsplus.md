# Документация к API "NEWSWeb"
NEWSWeb API это простой HTTP REST API для поиска новостей в режиме реального времени из разных вэб-ресурсов по заданным критериям.

---

## 1. Начало работы
Для начала работы с API необходимо пройти регистрацию на newsplus.com и получить API-ключ в личном кабинете.

Пример запроса:

```bash
curl -X GET "https://api.newsplus.com/v1/news?q=технологии&apiKey={API_KEY}"
```

Пример ответа:

```json
{
    "status": "ok",
    "totalResults": 523,
    "articles": [
        {
            "source": {
                "id": "dzen",
                "name": "Дзен"
            },
            "author": "Петров Петр",
            "title": "Пример описания API",
            "description": "Некоторое описание",
            "url": "https://www.dzen.ru/news/some-news-about-tech",

```

## 2. Аутентификация

Получить API-ключ в личном кабинете, в разделе "Безопасность".

Для выполнения запроса необходимо передать ключ одним из двух способов:
- в строке запроса, через параметр `apiKey`
- в заголовке HTTP-запроса `Authorization`

---

## 3. Endpoint

### Получение новостей по категориям

`GET /news/category/{category}`

Возвращает топ новостей по следующим доступным категориям:

- business - Бизнес и финансы;

- technology - Технологии;

- health - Здоровье и медицина;

- sports - Спорт;

- entertainment - Развлечения;

- science - Наука;

- general - Общие новости


### Поиск по источникам

`GET /sources`

Возвращает список доступных новостных источников.

### Поиск новостей по всем источникам

`GET /news`

Возвращает список новостей по критериям поиска. По умолчанию возвращает новости из всех источников за последние 5 лет.

Параметры запроса:
`q` - слово или фраза для поиска в заголовке или теле статьи;

`sources` - строка с источниками поиска статей, которые разделяются запятыми в случае нескольких источников;

`from` - дата самой старой статьи в ответе;

`to` - дата самой новой статьи в ответе.

### Пагинация

Каждый endpoint поддерживает пагинацию. Параметр `limit` задает количество возвращаемых элементов на странице, а параметр `page` - порядковый номер страницы результата запроса.

## 4. Коды ошибок

| Код HTTP | Код ошибки | Описание
| --- | --- | --- |
| `400` | `missing_parameter` | Отсутствует обязательный параметр (например, q или apiKey) 
| `401` |  `invalid_key` | Неверный или просроченный API-ключ
| `404` | `not_found` | Ресурс не найден 
| `500` | `server_error` | Внутренняя ошибка сервера

Пример ошибки:

```json
{
  "status": "error",
  "code": "invalid_key",
  "message": "Неверный API-ключ. Проверьте ключ в личном кабинете."
}
```

---

## 5. Советы по использованию

Для оптимизации запросов используйте конкретные категории вместо общего поиска.

Кэшируйте результаты на 5-10 минут для снижения нагрузки.

Всегда проверяйте поле `status` в ответе.

Используйте `page` и `limit` для работы с большими наборами данных.
