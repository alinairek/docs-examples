# Инструкция по объединению аккаунтов

## Для чего нужно объединение аккаунтов

Информация, указанная вами в разных аккаунтах, может быть объединена в единую карточку поиска Яндекса. Это помогает друзьям, коллегам или клиентам быстро и удобно находить информацию о вас, размещённую на разных площадках.

## Как работает объединение аккаунтов

Определить, что разные аккаунты принадлежат одному человеку, можно на основе **перекрёстных ссылок** или **подтверждённой привязки**.

1. Как работают перекрестные ссылки

 Вы указали ссылку на аккаунт ВКонтакте в аккаунте Telegram, и в аккаунте ВКонтакте добавили ссылку на тот же Telegram.
 
>**Совет:** Старайтесь делать ссылки взаимными. Если в Telegram вы указали ссылку на аккаунт ВКонтакте, добавьте ссылку на Telegram в аккаунт ВКонтакте. Это самый надёжный способ подтвердить поисковой системе, что оба аккаунта принадлежат вам.

2. Как работает подтверждённая привязка

 Вы входите в аккаунт одного сервиса через авторизацию другого. Например, при входе в аккаунт ВКонтакте вы выбираете "Войти через Госуслуги" и проходите авторизацию. После этого система понимает, что эти аккаунты принадлежат одному человеку.

Для более быстрого и простого варианта можно использовать привязку аккаунтов в **Яндекс ID**.

## Объединение аккаунтов в Яндекс ID

Привязка внешних аккаунтов к Яндекс ID упрощает их объединение, позволяет управлять доступами из одного аккаунта и повышает уровень безопасности.

Рекомендуется использовать ваш основной аккаунт Яндекса. Для этого необходимо выполнить следующие действия:

1. Перейдите в раздел [**Внешние аккаунты**](https://id.yandex.ru/security/external-accounts)

2. Выберите сервис, аккаунт которого хотите привязать в разделе **Привязать аккаунт** . 

3. Следуйте указаниям на экране.

##  Как отвязать внешний аккаунт

Чтобы отвязать внешний аккаунт:

1. Перейдите в раздел [**Внешние аккаунты**](https://id.yandex.ru/security/external-accounts)

2. Выберите аккаунт сервиса, который хотите отвязать.

3. Нажмите **Отвязать аккаунт**.

4. Подтвердите действие нажав **Отвязать**.

После отвязки аккаунта сервисы перестанут получать доступ к данным.

# 2. Интерфейс Яндекс.Учебника

В исходном варианте были выялены следующие недочеты:

1.	Много текста между призывом **Оставайтесь на связи** и кнопкой **Добавить**, из-за чего основная цель действия теряется.

2.	Фраза «дублировать» вызывает негативную ассоциацию у пользователя «еще больше навязчивых рассылок».

3.	Элемент **У меня нет другой почты** выделяется активнее кнопки **Добавить** и смещает фокус с целевого действия.

4.	Отказ от действия для пользователя более интуитивен и комфортен через кнопку закрытия формы, чем через отдельный текстовый элемент **У меня нет другой почты**.

5. Не предусмотрен чекбокс для подтверждения согласия на рассылку.

6. Не предусмотрен чекбокс для подтверждения согласия с правилами обработки данных.

Для нового варианта были внесены следующие изменения:

1. Текст сокращён и сфокусирован на ключевой ценности действия.

2. Добавлены чекбоксы для управления подпиской и подтверждения согласия на обработку данных.

3. Удален элемент **У меня нет другой почты** как избыточный.

4. Усилена визуально кнопка **Добавить**.

# 3. Практическое задание с curl

## Запрос curl

```bash
curl https://dictionary.yandex.net/api/v1/dicservice/lookup -G \
    -d key={MY_API_KEY} \
    -d lang=en-ru \
    -d text=sky \
    -d ui=ru
```
